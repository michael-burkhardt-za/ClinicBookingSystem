Running the demo
1. Start the API project first. It runs on https://localhost:7144
2. Start the Client project second. Debug breakpoints in the client will now attach.
3. No configuration changes are required.

This demo project illistrates the following.

Clean layering
Application-level validation
Global exception handling
DI registration done correctly
No deprecated packages
No over-engineering
